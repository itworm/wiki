# 从自己能用到别人敢用：Skill 的验证、版本与交接

作者：Zoe产品手记 ｜ 来源：人人都是产品经理

## 核心论点

一个 Skill 从"自己能用的"到"别人敢用的"，关键不在 prompt 写得多完整，而在于：失败记录、版本证据、交接规则三个要素是否齐备。

## 四层诊断框架

| 层 | 问题 | 解法 |
|----|------|------|
| 输入层 | 信息不足时继续装懂 | 开工闸门：先列已知/未知，确认后再继续 |
| 过程层 | 一步到位没有确认节点 | 拆成多个确认节点，错误回退到最近一步 |
| 输出层 | 结构与验收标准模糊 | 明确输出格式+质量检查项 |
| 验证层 | 改完凭感觉说"好多了" | 版本记录+复测+失败记录 |

## 关键方法

- **事实标签**：输出中区分"已确认"和"推断"，防止 AI 把推测写成事实
- **自查清单**：检查项必须具体可执行，不写"请你认真检查"
- **失败兜底**：定义"什么时候必须停下来"，比定义"怎么继续"更重要
- **版本记录**：每次只改一条规则，记录改了什么、为什么改、怎么验证
- **复测**：用相近但不同的任务测试，而非重复同一个案例

## 30分钟极速迭代

选小任务 → 跑 V1 → 找最影响使用的一个问题 → 判断属于哪一层 → 只改一条规则
