# 从自用到开源——个人知识库产品化实践

作者：Sean ｜ 来源：人人都是产品经理

## 三道坎：个人项目→终端产品

1. **用户画像缺失**——得服务一个不认识的人，不能只为自己设计
2. **个人印记剥离**——框架与内容分离：通用能力层（Skill/脚本/治理规则）vs 个性化配置层（身份/领域/演进方向）
3. **产品说明重构**——让新用户三分钟看懂"这是什么、和我有什么关系、第一步该做什么"

## 三个具体设计问题

| 问题 | 解法 |
|------|------|
| 框架和内容怎么拆？ | 两层解耦——通用能力层（不变）+ 个性化配置层（CLAUDE.md + ke-config.yaml） |
| 用户怎么定制不改源码？ | 配置分离——CLAUDE.md管AI行为，ke-config.yaml管知识库表现 |
| 安全隐患 | API Key明文→.env文件读取，发布前走一遍完整流程验收 |
