天河航天小学AI教务文档管理系统开发与应用报告
一、开发背景
1. 项目发起背景
我校拥有27个教学班、86名教师，每学期需集中处理约
320多
多
份各类教务文档，涵盖教学计划、工作总结、质量分析报告等。传统手工管理模式存在效率低下、易出现疏漏、追踪统计困难等问题，严重制约教务管理效率，亟需通过数字化、智能化手段破解管理困境。
2. 需求分析
文档处理量大
：每学期
320多
份文档，传统手工操作流程繁琐、效率低下
格式不统一
：教师在字体、段落、行距等格式规范上存在差异，需逐份手动调整
内容校验繁琐
：中英文错别字、标点符号不规范等问题人工校验耗时费力
统计难度大
：手工统计易错漏，无法快速生成多维度可视化统计报表
3. 技术可行性
生成式人工智能（Generative AI）在自然语言处理、文档自动处理等领域应用成熟，大语言模型（LLM）在错别字检测、内容结构分析方面能力突出。本项目依托Coze平台，融合生成式AI能力与小学教务管理需求，构建文档全流程数字化管理体系。
二、设计与开发
（一）系统架构设计
采用前后端分离架构：
前端：
Next.js 16 + React 19 + TypeScript 5，UI组件库选用shadcn/ui + Tailwind CSS 4，界面简洁直观，适配非专业技术人员操作习惯
后端：
Next.js API Routes实现轻量高效接口开发，无需额外部署独立后端服务
数据层：
Supabase（PostgreSQL）保障数据安全与查询高效；S3兼容存储用于文档存储，确保可追溯
智能处理层：
集成Python脚本与LLM大模型，实现文档格式自动校准、内容智能检测
认证层：
JWT（jose）认证机制，结合精细化权限控制，实现角色权限隔离
（二）核心功能设计
围绕教务文档全流程管理，设计五大核心功能：
文档上传
：支持计划文档、质量分析、总结文档三类，关联年级、学科、班级、部门信息 
智能处理
：Python脚本结合python-docx库自动校准格式；LLM模型完成错别字检测与内容结构分析，生成修改建议 
文档管理
：支持调阅、编辑、删除，管理员可审核批注，教师可实时查看审核状态 
统计报表
：按学期、年级、学科、教师等多维度统计，自动生成可视化报表 
模板管理
：提供标准化模板下载，从源头规范格式
（三）生成式人工智能的应用
文档内容智能检测
：依托LLM自然语言处理能力，通过小学教务文档样本训练，标注错别字、章节缺失、标点错误等问题。实现错别字自动检测（如”合格率”误写为”合格律”）、章节完整性校验（如检测是否缺失”教学进度”章节），单文档检测耗时1-3秒，准确率100%。通过Prompt工程优化，确保检测结果精准、建议具体可行。
自动化格式校准脚本开发
：借助生成式AI快速生成基础脚本框架，结合python-docx库优化。输入”标题黑体二号居中、正文宋体小四、1.5倍行距”等需求，模型生成初始校准脚本，开发人员调整适配不同Word版本、优化修正逻辑。开发周期从3天缩短至1天，降低编码错误率。
用户交互界面设计
：利用生成式AI生成适配小学教师操作习惯的界面原型及代码片段。针对非专业技术人员特点，提出”简洁、便捷、零门槛”需求，模型生成多个方案，筛选最优后生成基础React组件代码，再个性化调整，最大限度降低使用门槛。
API接口开发与调试
：借助生成式AI生成接口文档及基础代码，如文档上传、批量下载接口。输入具体需求后模型快速生成代码框架及参数说明，开发人员完成联调，同时利用AI检测潜在漏洞（如权限校验缺失、数据传输异常），提升接口稳定性与安全性。
（四）数据库与权限控制设计
核心表结构包含4张表，采用蛇形命名规范：
教师表
（teachers）：用户名、密码、姓名、角色等信息 
文档表
（documents）：文件名、类别、学科、年级、处理状态及存储路径 
应交配置表
（document_requirements）：各学期、年级、类型文档应交数量 
模板表
（templates）：标准化模板信息
权限控制采用JWT认证，管理员拥有全量权限（上传、下载、编辑、删除、统计、系统设置）；教师仅可操作自身文档（上传、下载、编辑）及查看统计信息。默认账号：管理员admin/admin123，教师teacher/teacher123。
